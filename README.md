# Transportation Data Pipeline & Mode Prediction
*A DSCI 510 Final Project*

This project outlines an end-to-end data pipeline: from fetching and cleaning massive external datasets, to training modern machine learning classification algorithms in order to predict human transportation mode choices (Walking, Driving, Public Transit).

#### Coding Agent Used: Google Antigravity. 

## Data Sources
The primary datasets used in this project originate from the National Household Travel Survey (NHTS) 2017 and the U.S. Census API. After processing, merging, and cleaning, the final data shape evaluated by the models consists of **`923,572` rows and `221` columns**. 

Below is a detailed breakdown of the raw data sources ingested by the pipeline:

| Data Source Name | Source URL | Type | List of Fields (Selection) | Format | Extracted via Python? | Estimated Data Size |
|---|---|---|---|---|---|---|
| **NHTS 2017 Household Data** <br> *(Household-level socioeconomic characteristics)* | [nhts.ornl.gov](https://nhts.ornl.gov/) | File | `HOUSEID`, `HHSIZE`, `HHVEHCNT`, `HHFAMINC`, `HHSTATE`, `NUMADLT`, `YOUNGCHILD` | CSV | Yes | ~100,000 records |
| **NHTS 2017 Trip/Person Data** <br> *(Individual travel behavior and mode choice)* | [nhts.ornl.gov](https://nhts.ornl.gov/) | File | `HOUSEID`, `TRAVDAY`, `TRIPID`, `TRPTRANS` (mode), `TRPMILES`, `TRVLCMIN` | CSV | Yes | ~500,000 records |
| **U.S. Census ACS Data** <br> *(State-level demographic and income data)* | [api.census.gov](https://api.census.gov/data.html) | API | `median_income`, `population`, `employment_status`, `state_code` | JSON | Yes | ~1,000+ records |

*Post-Processing outputs generated by the pipeline:*
- **`x.csv` (Features)**: The final synthesized array of comprehensive categorical variables describing survey respondents based on the sources above.
- **`y.csv` (Target Variable)**: Contains the final transportation mode chosen by the respondent corresponding to each trip.

---

## Data Retrieval & Cleaning Pipeline
The foundation of this project is built upon two Python scripts designed to construct the dataset automatically:

* **`api_access.py`**: Connects to the external API, sends requests to retrieve raw data, and safely saves the response locally. Utilizing `requests`, `pandas`, `json`, and `os`.
* **`clean_nhts_data.py`**: Loads the raw dataset to clean and preprocess the information. It handles missing/invalid values, renames columns, filters relevant records, formats data types, and outputs the refined dataset (e.g., `x.csv` and `y.csv`) ready for modeling.

---

## Results & Modeling
The robust execution pipeline outputs both statistical benchmarks and highly visual exploratory analytics:
1. **Algorithmic Accuracy Benchmark**: Through evaluating `XGBoost`, `Logistic Regression`, and `Decision Tree Classifier` models, the pipeline organically extracts the highest performing algorithm into a summarized table (`model_results.csv`).
2. **Behavioral Feature Extraction**: Using both an aggregated Random Forest and an isolated Decision Tree, the pipeline strictly calculates and plots exactly what attributes dictate travel methods the most (`top_10_features_rf.png` & `top_10_features_dt.png`).
3. **Regional Transit Visualization**: Pandas groupby techniques combined with Matplotlib render how different geographical Census Regions fundamentally dictate mass transit vs. car adoption (`travel_modes_by_area.png`).

## Installation

To run this data science pipeline locally, you will need Python 3 and standard data manipulation packages. 

Install the required dependencies using pip:
```bash
pip install pandas requests scikit-learn xgboost matplotlib seaborn
```

*Note: There are no specific `.env` keys required for this local machine learning approach. If you intend to skip the API pipeline, ensure that your cleaned datasets (`x.csv` and `y.csv`) are placed directly into the root directory before running the ML scripts.*

## Running Analysis

The project is split into distinct functional scripts for clarity. From the root directory, run the following commands sequentially to reproduce the entire pipeline from scratch:

**1. Fetch and Preprocess Data**
If you need to construct the baseline data from the external source:
```bash
python api_access.py
python clean_nhts_data.py
```

**2. Run the Core ML Pipeline** 
Trains the models, handles one-hot encoding, prevents outlier errors via thresholding, and exports testing accuracy metrics:
```bash
python run.py
```
*Results will appear cleanly inside the root directory as `model_results.csv`.*

**3. Isolate Feature Importance** 
Generates top-10 variable bar graphs utilizing tree-based architectures to find what really drives human travel behaviors:
```bash
python rf_importance.py
python dt_importance.py
```

**4. Generate Exploratory Graphics**
Automatically renders the correlation matrix and the stacked geographic transit distribution bars:
```bash
python visualization.py
```

## Methods Explored 
- **XGBoost**: Highly resilient non-linear tabular modeling. 
- **Logistic Regression**: Linear probabilistic distance models standardized via `StandardScaler`.
- **Decision Trees & Random Forests**: Rule-based node splitters capped at specific depths (`max_depth=20`) to prevent data overfitting on the large dataset.
